## 1. Crystallised problem

## Problem name
AI-Assisted Solutioning Estimator

## Crystallised statement
Solo founders operating within a 4WRD delivery cycle lack a fast, repeatable way to estimate the effort required for a solutioning engagement from free-text problem input alone. There is no tool that infers likely failure modes from that input and returns dual-mode estimates — conventional and 4WRD — expressed in person-days, without dependency on historical data. This tool must produce its output as structured markdown, consumable immediately in Notion (primary) and Google Doc (secondary), so that the estimate artefact is ready for use without reformatting. The estimator must work for a solo founder with no team and no prior engagement history to draw on.

## In scope
- Acceptance of free-text problem input as the sole required input for estimation
- AI inference of failure modes from the free-text input
- Generation of dual-mode estimates: conventional delivery estimate and 4WRD delivery estimate, both expressed in person-days
- Structured markdown as the sole output format
- Compatibility with Notion (primary) and Google Doc (secondary) without reformatting
- Operation without historical data or a team

## Out of scope
- Integrations with project management or time-tracking platforms
- Multi-user or team-based estimation workflows
- Estimation methods that require historical engagement data as input
- Output formats other than structured markdown
- Post-estimate tracking or actuals reconciliation

4. The markdown renders correctly in Notion without any manual reformatting
5. The same markdown renders correctly in Google Doc without any manual reformatting
